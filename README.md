Unity 기반 턴제 전략 시스템을 구현한 프로젝트

A* 경로 탐색과 Coroutine을 활용한 비동기 처리로  
경로 탐색 과정에서 발생한 프리징 문제를 해결하는 데 초점을 두고 개발

시연 영상: https://youtu.be/IxUP-Wv2HZM

작업기간 (빌드기준)
  - 기술 탐색 및 검증 과정 2023.02.09 ~ 03.14
  - 리소스 수집 03.14 ~ 30
  - 게임 환경에 맞게 코드 수정 03.30 ~ 04.12
  - 테스트 및 밸런스 조정 04.12 ~ 15

개발 방식
  - 게임 구상 및 레퍼런스 조사 (게임 라이프타임, UI구성 등등)
  - 기술 탐색 -> 검증 -> 적용 반복

구조 설명
  - 타일, 아군, 적군 오브젝트를 관리하는 Container Script를 구성하여 탐색 비용을 줄이고 참조를 단순화
  - 각 Manager를 통해 Container에 접근하도록 구조를 분리하고, 전역 접근이 필요한 부분은 Singleton으로 구성하여 제어 흐름을 단순화
  - 게임 입력 및 흐름 제어는 cursorComp를 중심으로 처리하여 플레이어 인터랙션을 일관되게 관리
 
주요 문제 해결 사례
    문제 상황
  - A* 기반 경로 탐색 시, 이동 거리가 길어질수록 프레임 드랍 및 프리징 발생
    
    시도 1
  - 탐색 함수를 Coroutine으로 변경하여 노드 탐색마다 yield 적용
  → 프리징은 완화되었으나, 탐색 도중 이동 가능 범위가 실시간으로 변경되는 문제 발생
  
    해결 방법
  - 턴 시작 시 모든 캐릭터의 이동 가능 노드를 사전 계산
  - 플레이어 입력을 제한하여 상태 변화 중 데이터 일관성 유지
  - 입력 제한 동안 UI로 로딩 상태를 표시하여 UX 보완

    결과
  - 프리징 문제 해결 및 안정적인 턴 기반 동작 확보
  - 시스템 안정성과 사용자 경험을 동시에 고려한 해결 방식 적용
